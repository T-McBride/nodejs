Workflow

High Level This Demo Real World
1 A loan event triggers a web call from ICE to the client's endpoint Simulate this event with a manual POST request
e.g. curl --header "Content-Type: application/json" --request POST -d '{"message":"too short"}' http://localhost:3000/webhook/fevent		 Subscribe to a webhook, define your trigger, data, and receiving endpoint
2 Call is received and validated Calls for this demo are received on port 3000 and must have a data element of message and longer than e chars Data would be vaalidated for valid request, needed data provided, authorized user, known ip, etc
3 Validated requests can be 'handed off' to other complex processes For this demo, to simulate a secondary process, fake data is created and sent to a seperate webserver running on port 8088 Users can create their own complex processes.
e.g. A custom round robin that employess specific client rules and incorporates PTO data
This is also where versioning could take place
4a In addition to any internal process data can also be exchanged with ICE and other vendors at this step The fake data mentioned previously simulates what can be returned by these processes Return requests from other webcalls would be parsed to determine next steps in a compoud process
4b Results from all other processes can be handled here This demo page of this app publishes data received from the websocket server running on port 8088 This could be internal apps to show data, write to sql, send notificaitons, etc
